# Einleitung

myclaim ist eine leistungsstarke Web-Anwendung die Schadensmanagement in Zusammenarbeit
mit verschiedenen Partnern/Akteuren wie Werkstätten, Versicherer, Hersteller, Sachverständige
ermöglicht, unter der Einhaltung eines vordefinierten und anpassbaren Arbeitsablaufs.
Dabei kann ein bestimmter Akteur in einem individuell anpassbarem Regelwerk, mehreren
Partnern bestimmte Vorgaben festlegen die dann automatisch im Arbeitsablauf abgeprüft
werden und somit die Einhaltung dieser Vorgaben garantieren.

Des weiteren bietet myclaim in einer leistungsstarken XML und CSS Entwicklungsumgebung die Möglichkeit, individuell anpassbare Layouts bzw. Arbeitsvorlagen
(sogenannte Templates) nach Kundenwunsch zu erstellen und diese nachträglich anzupassen.

myclaim beinhaltet mitlerweile sehr viele Netze. Das bedeutet um z.B. in ein bestimmtes
Netz einen neuen Vorgang per Webservice anlegen zu können, ist es u.U. nötig diverse
andere Parameter oder Funktionen zu kennen.

Hierzu empfiehlt es sich eine netzspezifische myclaim Schulung in Anspruch zu nehmen.

| Begriff | Bedeutung |
| --- | --- |
| contract | Ein Geschäftsauftrag (Vorgang) innerhalb eines Netzes (Kostenvoranschlag, Glas Berechnung, Fahrzeug Instandsetzungskalkulation etc.) |
| template | Eine individuell anpassbare Arbeitsbereich die diverse Daten,- bzw. Eingabefelder beinhaltet |
| partner | Ein definierter Kunde/Anwender, der an der Verarbeitung eines bestimmten Auftrags unter einer bestimmten Rolle teilnimmt. |
| network | Ein Netzwerk das individuelle Prozesse / Arbeitsabläufe, Status sowie diverse Rollen und Berechtigungen umfasst. |
| customer role | Rolle eines Benutzers oder Partners in einem Auftrag (z.B. Werkstatt, Versicherer etc…) |
| user | Benutzer des Auftrags |
| Dossier | Aktenzeichen Vorgang siehe [Aktenzeichen Vorgang (Dossier)](../../vxs/aktenzeichenvorgan/aktenzeichenvo-1371.md) |

Myclaim Schnittstellen

Mit Hilfe der myclaim Schnittstellen habe Sie generell die Möglichkeit standardisiert
über die SOAP Technologie, Zugang zu folgenden Funktionen:

- Detaillierte Informationen eines oder mehrerer Aufträge

- Erstellung eines neuen oder Aktualisierung eines vorhandenen Auftrags

- In einem Auftrag die Zuweisung oder Entnahme eines Partners innerhalb einer bestimmten
Rolle

- Statusänderungen eines vorhandenen Auftrags

- Auflistung aller Partner innerhalb eines Netzwerks

- Auflistung aller Anhänge eines Auftrags inkl. aller vorhandenen Auftragsvorlagen

- Austauschen von Nachrichten zwischen verschiedenen Akteuren innerhalb eines Auftrags
