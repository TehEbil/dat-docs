# Basis

Gegeben seien E1 bis E6, mit den folgenden logischen Begriffen:

```
E1 INCLUDES AND(AND(E2))
E2 INCLUDES AND(AND(E3))
E3 REQUIRES AND(AND(E4))
E4 EXCLUDES AND(AND(E5))
E5 EXCLUDES AND(AND(E6))
E6 EXCLUDES AND(AND(E5))
```

Die folgende Tabelle listet Kombinationen von Aufrufen von validateConfiguration und die entsprechenden Ergebnisse auf:

| Im Aufruf angegebene Ausstattung (jede Zeile in einer Reihe bedeutet einen Aufruf) | Fehlercode | Fehlerdetails | Begründung |
| --- | --- | --- | --- |
| [] | 0 |  |  |
| E1 E2 E1, E5 E1, E6 | ERR\_REQUIRED\_MISSING | [E4] | E4 erforderlich durch E3, aber nicht spezifiziert |
| E4 E5 E6 | 0 |  |  |
| E1, E2 E1, E2, E4 E1, E2, E5 | ERR\_ALREADY\_INCLUDED | [E2] | E2 angegeben, obwohl in/durch E1 enthalten |
| E1, E2, E3 | ERR\_ALREADY\_INCLUDED | [E2, E3] | E2 und E3 angegeben, obwohl in/durch E1 enthalten |
| E1, E4 E2, E4 E3, E4 | 0 |  |  |
| E1, E4, E5 | ERR\_EXCLUDED\_SPECIFIED | [E5] | E5 durch E4 ausgeschlossen, aber dennoch angegeben |
| E4, E6 | 0 |  |  |
| E5, E6 E6, E5 | ERR\_EXCLUDED\_SPECIFIED | [E6] | E6 durch E5 ausgeschlossen, aber dennoch angegeben |
| E7, E8 | ERR\_UNKNOWN\_ITEM | [E7, E8] |  |
