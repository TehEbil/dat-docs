# Basis

Gegeben sind O1 bis O6, mit den folgenden logischen Begriffen:

```
O1 INCLUDES AND(AND(O2))
O2 INCLUDES AND(AND(O3))
O3 REQUIRES AND(AND(O4))
O4 EXCLUDES AND(AND(O5))
O5 EXCLUDES AND(AND(O6))
O6 EXCLUDES AND(AND(O5))
```

Die folgende Tabelle listet Kombinationen von Aufrufen von validateConfiguration und die entsprechenden Ergebnisse auf:

| Im Aufruf angegebene Optionen (jede Zeile in einer Reihe bedeutet einen Aufruf) | Fehlercode | Fehlerdetails | Begründung |
| --- | --- | --- | --- |
| [] | 0 |  |  |
| O1 O2 O1, O5 O1, O6 | ERR\_REQUIRED\_MISSING | [O4] | O4 erforderlich durch O3, aber nicht spezifiziert |
| O4 O5 O6 | 0 |  |  |
| O1, O2 O1, O2, O4 O1, O2, O5 | ERR\_ALREADY\_INCLUDED | [O2] | O2 angegeben, obwohl in/durch O1 enthalten |
| O1, O2, O3 | ERR\_ALREADY\_INCLUDED | [O2, O3] | O2 und 3 angegeben, obwohl in/durch O1 enthalten |
| O1, O4 O2, O4 O3, O4 | 0 |  |  |
| O1, O4, O5 | ERR\_EXCLUDED\_SPECIFIED | [O5] | O5 durch O4 ausgeschlossen, aber dennoch angegeben |
| O4, O6 | 0 |  |  |
| O5, O6 O6, O5 | ERR\_EXCLUDED\_SPECIFIED | [O6] | O6 durch O5 ausgeschlossen, aber dennoch angegeben |
| O7, O8 | ERR\_UNKNOWN\_ITEM | [O7, O8] |  |
